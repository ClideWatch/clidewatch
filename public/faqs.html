What is a clIDE - and why would i need a guide ?

The word clIDE is a contraction of cloud-hosted-integrated-development-environment; which descibes a class of gadgets, of which serveral instances have sprung up over the past couple of years. You probably wouldn't want to have to say that too many times; so we coined this term for you. You're welcome!

Generally speaking, a clIDE is a website that offers access to a virtual computer, that software developers can use to build and test their programs using their web-browser. They are generally best suited for web development; but some are fully functional computers systems that can run nearly any code. Some of them even allow multiple people to type code into the same file at the same time and see the others changes as they type in real time. Add in a chat and voice channel and these make for an excellent platform for collaboration between project team members, as well as students and teachers.

There are several of these out there now to choose from and more popping up all the time; so this site was created to attemp to keep track of them all and to make impartial comparisons and reviews in a central location to help newcomers decide which is the best fit for their needs. We use our favorites of these tools regularly and have tested the ones you'll find reviewed here thoroughly (well, a little thoroughly - seems there's always a new one to try).

How does clIDE-guide rate/compare clIDEs ?

All of the the hosted services listed here offer a free tier and that is the basis of comparisson. These are the criteria we are currenty tracking:

  • Memory - This is the total VM memory shared by all kernel and user processes.
  • Disk - This is the actual available empty disk space you get on a fresh VM and not the amount alloted to the entire VM (much of which is occupied by the OS).
  • Private - Some of the hosted services allow all users to keep all of of their data private while others are free only for open source projects and reserve this feature for paid subscriptions only.
  • Colaboration - Regardless of the privacy status most service allow some number of collaborators on your projects. In all cases though, even if the project is publicly readable; you are the one to decide who may edit your files. Some of them even have a real-time shared editing feature, whereby multiple people can edit the same code simultaneously and see each other's typing as they type (very nifty). This latter group are the ones marked with a 'yes' for this criteria.
  • Community - Some of the hosted services have integrated community/chat interaction.
  • Sudo - Some of the hosted services offer fully priveleged VM logins.
  • Ssh - Some of the hosted services allow you to access the VM using an ssh client.
  • Ssl - Some of the hosted services feature an https url into your webservers.
  • Hosted - Some of these web-based IDE's are available 'in the cloud' only and some others require you to host on your own server; and there are yet others that offer both options.
  • Source - Some of these web-based IDE's are free and open-source; allowing you to run your own cloud IDE service if you desire.
